Why Builder

Built for operability, not just output.

Builder is opinionated about runtime control. That means policy, verification, and lineage are first-class runtime concerns instead of model-side conventions.

01 Runtime ownership

Completion is a runtime decision.

Builder does not stop at “the model answered.” It routes the turn, constrains tool scope, captures work state, and only closes after verification proves the result.

  • Intent classification and routed execution plans
  • Tool policy that can gate shell, write, and network surfaces
  • Verification-aware completion instead of success-shaped guesses
02 Operational backend

PostgreSQL 18 + pgvector under the hood.

State, retrieval, vector search, and runtime memory share one operational backend instead of being scattered across ad-hoc local files.

One backend, one truth source
03 Delegated lineage

Child runs stay linked to the parent objective.

Delegation packets, overlays, mission IDs, and recovery routes preserve why a task existed and how it should be re-entered if it drifts.

Runtime contract

What the runtime actually owns.

Builder’s public story is simple: the runtime stays responsible for the hard parts.

01

Classify the turn

Understand whether the request is planning, implementation, research, delegation, or review.

02

Constrain tool policy

Pick the execution profile that defines shell, write, network, and evidence boundaries.

03

Record work state

Persist lineage, state cards, packets, and recovery information while the work unfolds.

04

Adjudicate completion

Completion closes only after validation confirms that the runtime contract was actually met.

Quick start

Pick the shortest path into Builder.

Compose for the full stack, binary for the shortest install, image for a container-first workflow.

Compose

Run the full runtime stack

The best default if you want PostgreSQL + Builder immediately.

curl -O https://raw.githubusercontent.com/agnusdei1207/builder/main/compose.yaml
docker compose up -d

BUILDER_PROJECT_DIR=$(pwd) docker compose run builder
Binary

Download the CLI directly

Fastest route when you just want the executable in hand.

curl -LO https://github.com/agnusdei1207/builder/releases/latest/download/builder-x86_64-unknown-linux-musl
chmod +x builder-x86_64-unknown-linux-musl
./builder-x86_64-unknown-linux-musl --help
Docker image

Use the published image

A container-native path that still keeps your project mounted locally.

docker pull agnusdei1207/builder:latest
docker run --rm -it -v "$(pwd):/workspace" -w /workspace agnusdei1207/builder:latest
