Dynamic Agent Lifecycle

Agents are not static. Based on your intent, the runtime spawns specialized agents, scales them when tasks grow complex, and dissolves them when their work is done.

The harness manages agent creation, expansion, delegation, and cleanup — all driven by the runtime's orchestration layer.

builder
// "implement login API"
spawn researcher agent ✓
spawn builder agent ✓
[expand] complexity detected
[delegate] test-runner spawned
[delegate] reviewer spawned
[verify] all agents reporting
dissolve test-runner ✓
[dissolve] reviewer ✓
TASK COMPLETED · 2 agents dissolved
On-Demand Spawning
Agents spawn only when needed. The harness monitors task complexity and triggers new agents dynamically.
Auto-Expansion
When tasks grow complex, the harness expands the agent team. Sub-agents delegate work and report back.
Graceful Dissolution
Agents dissolve when their purpose is served. No lingering processes, no wasted resources — pure task-based lifecycle.

Agent Lifecycle Phases

PhaseTriggerAction

Quick Start

01 docker compose 
# PostgreSQL 18 + Builder
curl -O https://raw.githubusercontent.com/agnusdei1207/builder/main/compose.yaml
docker compose up -d

# Run builder harness in your project
BUILDER_PROJECT_DIR=$(pwd) docker compose run builder
02 direct install 
# Download binary
curl -LO https://github.com/agnusdei1207/builder/releases/latest/download/builder-$(uname -m)-unknown-linux-musl

chmod +x builder-*
./builder-* --help
