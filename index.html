runtime-owned coding agent CLI

Builder keeps the runtime in charge.

Builder is a local-first coding agent CLI where the runtime, not the model, owns routing, tool policy, verification, completion adjudication, and delegated-run lineage. PostgreSQL 18 + pgvector is the operational backend for control-plane state, retrieval, and memory.

Execution contract

What the runtime is actually responsible for

Builder treats routing, policy, verification, and completion as runtime work. That is the core architectural difference from model-led agent shells.

01 classify

Classify the request

Intent classification and request routing decide whether the turn is planning, implementation, review, or a general task.

02 constrain

Constrain tool policy

Execution strategy and dynamic profile control shell, write, and network permissions before any tool call happens.

03 record

Record work and lineage

Objectives, state cards, evidence, recovery routes, and delegated child-run lineage are written into the operational store.

04 adjudicate

Verify and adjudicate completion

A polished final answer is not enough. Builder closes the loop only after verification and completion proof succeed.

Quick start

Run Builder the public way

Choose the path that matches how you want to operate Builder: the public compose stack, a direct binary download, or the published Docker image.

compose

PostgreSQL + Builder

Good default when you want the full public runtime stack with the published compose file.

curl -O https://raw.githubusercontent.com/agnusdei1207/builder/main/compose.yaml
docker compose up -d

BUILDER_PROJECT_DIR=$(pwd) docker compose run builder
binary

Download the CLI

Pull the latest public binary from GitHub Releases when you do not want the full compose stack.

curl -LO https://github.com/agnusdei1207/builder/releases/latest/download/builder-x86_64-unknown-linux-musl
chmod +x builder-x86_64-unknown-linux-musl
./builder-x86_64-unknown-linux-musl --help
docker image

Use the published image

Run Builder in a container without copying the public compose file into your project first.

docker pull agnusdei1207/builder:latest
docker run --rm -it -v "$(pwd):/workspace" -w /workspace agnusdei1207/builder:latest
